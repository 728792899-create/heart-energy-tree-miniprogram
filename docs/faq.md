# 常见问题与排障

本页同时面向使用者和维护者。先确认问题属于产品边界、本地环境、云端部署、内容安全、数据运维还是 CI，再按对应步骤处理。

## 产品边界

### 这是公共情侣社交平台吗？

不是。它只服务一段固定两人关系，不会擅自改成公共多租户产品，也没有用户发现、公开动态、排行榜或跨关系搜索。

### 能量币可以充值、提现或投资吗？

不能。能量币只是两个人约定的内部记账单位，不代表存款、利息、收益或金融资产。项目不接真实支付，不提供充值、自动转账或投资能力。

### 心愿金怎样兑现？

打卡者提交申请后，赞助者确认并在线下手动兑现，再回到小程序标记状态。所有页面都必须继续使用“手动兑现”表述。

## 本地运行

### 为什么仓库没有 `project.private.config.json`？

它属于微信开发者工具的个人设置，可能携带本机状态，因此不会提交。公开仓库提供 `project.private.config.example.json`；需要时复制为私有文件，实际文件继续保持忽略。

### 为什么普通测试不需要 Remotion 输出？

`npm test` 只校验可提交源码和本地 poster。真实渲染由 `npm run motion:smoke` 单独完成，输出写入已忽略的 `motion-studio/out/`。

### 怎样运行完整公开仓库检查？

运行 `npm run ci`。它依次执行测试、共享代码同步、JavaScript 语法、云函数依赖、包体预算和 `check:docs` 文档检查。

## 云端与身份

### 页面提示云函数版本过旧怎么办？

先部署 `energyTree` 云函数，再调用 `queryDashboard` 核对响应 `buildTag`。客户端与云端 buildTag 不一致时，不要继续用旧云函数做验收证据。

### 为什么不能相信客户端传来的角色或 OPENID？

客户端输入可以被修改。生产权限只接受 `getWXContext().OPENID` 得到的可信 OPENID，并在云函数内继续校验关系成员和角色。

### `npm run check:shared` 失败代表什么？

共享主版与云函数部署副本发生漂移。只修改客户端可信源，然后运行 `npm run sync:shared` 生成部署副本；不要分别手改两份业务规则。

### 可以直接从小程序写业务集合吗？

不可以。业务写操作必须经过云函数。数据库规则中的业务集合 `write` 应保持 `false`，由可信服务端执行事务、鉴权和审计。

## 内容安全与图片

### 文字内容安全怎样处理？

云函数在写入前调用官方文字检查；服务不可用时失败关闭，不把未经检查的内容当作成功结果。

### 图片内容安全为什么是异步的？

提交图片后先登记 `traceId` 和业务引用，再等待微信 `wxa_media_check` 回调。风险结果会隐藏业务图片引用、删除对应云文件并写审计记录。

### 回调一直没有到怎么办？

核对平台消息推送配置、环境、回调路由和 `mediaCheckTasks`。没有平台权限时标记“外部权限阻塞”，不能用本地模拟测试冒充真实回调。

## 动效与声音

### 动效加载失败时会怎样？

项目保留三级降级：远程 MP4 → 本地 poster → 原生 WXML/WXSS 静止可读状态。远程视频不是发布必需项。

### 怎样减少动效或关闭声音？

“我的”页提供声音和简化动效开关，设置持久化。简化动效跳过远程视频，声音关闭不影响任何业务动作。

## 数据权利与恢复

### 当前有一键解绑或删除账号吗？

没有开放自助入口。关系解除、数据导出、数据删除和异常账号恢复属于人工高风险流程，必须进行双人身份复核、备份和审计。

### 数据删除包含照片吗？

包含。先冻结新写入，导出必要数据，删除业务引用和云存储文件，再验证临时 URL 已失效。详细顺序见 [隐私与数据生命周期](privacy-data-lifecycle.md)。

### 迁移失败可以直接清库重来吗？

不可以。先验证备份，使用幂等迁移并记录版本；失败时恢复到独立集合核对，再按审计流程回滚，不能清空现有云数据。

## CI 与文档

### `check:docs` 检查什么？

它检查 Markdown 本地链接、图片替代文字、JPEG/PNG 有效性、单图和总预算，以及页面目录是否覆盖 `app.json` 的全部路由。

### 图片预算失败怎么办？

先裁掉开发者工具外框，再使用合适的 PNG/JPEG 压缩；文档图片不应移入小程序主包。不要为了通过预算降低到文字不可读。

### GitHub Actions 通过是否等于已经发布？

不等于。CI 可以证明公开源码可复现，但不能证明云函数已部署、真实内容安全回调已送达或双账号真机流程已通过。

仍未解决时，按 [部署清单](deployment-checklist.md)、[真机验收表](device-acceptance.md) 和 [Release Checklist](release-checklist.md) 收集脱敏证据。
