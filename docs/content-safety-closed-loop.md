# 图片内容安全异步处置闭环

## 处理范围

小程序在提交打卡照片、头像、奖品图片和情侣信笺图片时，先调用微信 `mediaCheckAsync`，再把 `traceId`、可信 `openid`、关系、图片 `fileId` 和业务后端写入仅供云函数使用的 `mediaCheckTasks` 集合。

微信推送 `wxa_media_check` 结果后，`energyTree` 云函数按 `traceId` 找到待检任务：

- `pass`：保留图片，任务标记为 `pass`。
- `risky` 或 `review`：按 `fileId` 反查当前引用，清空图片字段并标记 `mediaHidden`/`hidden`，写入安全审计，然后尽力删除云存储文件。
- 图片已不再被业务记录引用：任务标记为 `orphan`，仍尝试删除对应云文件。
- 同一 `traceId` 重复推送：只有 `pending` 任务会处理，已完成任务直接跳过。

打卡、头像和奖品图片位于 `appStates/main`；信笺图片位于独立的 `coupleMessages` 与双方收件箱投影。两类后端分别处置，`mediaCheckTasks` 不进入公开快照集合。

风险图片下架不会自动冲正已经入账的能量币、余额、心愿金或兑换记录，避免异步内容结果造成财务状态不一致。系统只隐藏图片并留下审计记录，由赞助者根据实际情况人工决定后续业务处理。

## 微信平台必须人工配置

代码提交本身不会启用异步结果推送。发布人员必须在微信公众平台完成以下操作：

1. 进入“开发 → 开发管理 → 消息推送”（具体入口可能随平台调整）。
2. 将 `wxa_media_check` 事件配置为“云函数接收”。
3. 选择 `energyTree` 云函数作为接收函数。
4. 确认数据库集合 `mediaCheckTasks` 只允许云函数读写，小程序端禁止直接读取和写入。
5. 在获得明确发布授权后重新部署 `energyTree` 云函数；本仓库脚本不会自动上传或部署。
6. 使用合规测试账号完成“上传 → 业务提交 → 异步命中 → 自动隐藏 → 审计可见”的真机验收。

不要在验收记录中保存 `OPENID`、邀请 token、二维码、Cookie、私人头像、打卡照片或信笺图片。只记录脱敏后的时间、动作、`traceId` 后四位、处理状态和结论。

## 体验版合法内容误拦回归清单

以下项目必须在体验版真机环境人工执行，离线测试不能替代微信官方接口返回：

| 动作 | 合法测试内容 | 预期 |
| --- | --- | --- |
| `submitCheckIn` | 正常运动照片与日常运动备注 | 文字不误拦；图片异步返回 `pass`；照片保留 |
| `updateProfile` | 普通昵称与正常头像 | 文字不误拦；头像异步返回 `pass`；头像保留 |
| `sendCoupleMessage` | 日常情侣文字与正常生活图片 | 文字/图片不误拦；双方可查看 |
| `saveRewardItem` | 正常奖品名称、描述与图片 | 文字不误拦；图片异步返回 `pass`；奖品图保留 |

每项至少记录：动作、测试时间、`msgSecCheck.result.suggest`、`mediaCheckAsync.traceId` 后四位、回调 `suggest`、最终任务状态以及是否误拦。若出现误拦，停止发布，保留脱敏错误码并核对 `scene`、内容类型和微信平台能力配置。

## 风险闭环验收

使用专门的合规测试素材验证以下结果，不要使用真实用户隐私图片：

- 风险打卡图：`photoFileId` 和 `photoPath` 被清空，`mediaHidden=true`，产生 `mediacheck.risky.autohide` 审计。
- 风险头像：`avatarFileId` 和 `avatarUrl` 被清空，保留文字头像降级。
- 风险奖品图：`imageFileId` 和 `imageUrl` 被清空，页面使用既有兜底图。
- 风险信笺图：源消息和双方收件箱均 `hidden=true`、`imageFileId=''`，显示“该图片已被内容安全隐藏”。
- 重复回调不会重复删除文件或追加审计。
- 回调处理失败时任务保持 `pending`，不得把尚未完成的处置误记为通过。
